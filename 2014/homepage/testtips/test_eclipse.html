Test i Eclipse

Denna sida på svenska This page in English This page in English

Nedan beskrivs test med hjälp av JUnit 4 och med hjälp av Makefile i Eclipse.

Test med hjälp av JUnit 4

För dem som skriver sina program i Eclipse är det naturligt att utforma test med hjälp av JUnit. I denna fil finns ett exempel på hur man kan göra.

I filen finns ett enkelt testfall (testFriend) som läser en av de små infilerna och jämför resultatet med korrekt utdata. Liknande test kan skrivas för övriga testfiler. Det finns också en metod, testAll, som läser kör test för alla testfiler. Ni kan undvika att den körs från början genom att kommentera bort den eller genom att ta bort annoteringen @Test.

Testmetoderna i filen dirigerar om utdata från det program som körs till en fil. Denna fil jämförs sedan med innehållet i den fil som innehåller korrekt resultat.

Testprogrammet är skrivet med följande utgångspunkter:

  • Testfilerna (indata och korrekt utdata) finns i katalogen /usr/local/cs/edaf05/lab1. Om ni placerat dem någon annanstans så måste ni ändra definitionen av TESTDATA_DIR
  • Programmet som testas finns i klassen GS. Ändra här till ert eget namn på klassen.
  • Programmet som testas är skrivet så att det läser indata från en fil, som ges som argument till programmet (placeras i args[0] vid anrop av main-metoden)

Test med hjälp av Makefile

Här finns en makefile som kan användas i Eclipse. Placera filen (med namnet Makefile) i projektkatalogen för laboration 1.

Om ni inte använt makefiler tidigare kan det vara en bra idé att läsa lite om sådana. Det finns massor av information på nätet.

Den givna makefilen innehåller två s.k. targets med namnen test och testall. Det första kör programmet (GS) på två av de små indatafilerna. Här kan ni fylla på med fler test efter hand genom att lägga till nya rader. Tänk på att varje rad då skall inledas med tabulatortecken.

När man kör makefilen skriver den ut vilka kommandon som utförs. Testen är utformade så att de kör programmet med vissa indatafiler och jämför utdata från programmet med korrekt utdata som finns på en fil (med hjälp av kommandit diff). Om det finns skillnader skrivs detta ut. Ett lyckat test karaktäriseras alltså av att när makefilen körs blir de enda utskrifterna uppgifter om vilka kommandon som utförs.

För att använda makefilen i Eclipse måste vi tala om var det program som kan köra makefiler finns. För att t.ex. köra den med target test gör man så här:

  • Markera makefilen i projektkatalogen.
  • I Run-menyn: välj External Tools --> External Tools Configuration
  • Markera Program
  • Klicka på ikonen med ett +-tecken upp till vänster för att skapa en ny konfiguration. En konfiguration med namnet New_configuration dyker då upp i listan med rubriken Program
  • Markera den nya konfigurationen
  • Fyll i fältet Name med ett lämpligt namn på konfigurationen t.ex. make test
  • I fältet Location skall man fylla var det externa verktyget som skall köras finns. Ange /usr/bin/make
  • Fyll i fältet Working Directory med ${project_loc}
  • I fältet Arguments skall det argument som skall skickas till progrmmet make skrivas. Ange här: test
  • Klicka på knappen Apply
  • Klicka på Run om du vill prova att köra konfigurationen direkt

Nu har en ny konfiguration som motsvarar att köra kommandot 'make test' från tangentbordet skapats.

När ni vill köra den igen: Gå till Run menyn, välj ExternalTools. Här dyker konfigurationen make test nu upp som ett alternativ att välja.

Det finns ytterligare ett target i Makefilen, testall. Här körs alla de testfiler som finns i testkatalogen. För att köra det upprepar man hela proceduren ovan, men med nytt namn (t.ex make testall) och nytt innehåll i fältet Arguments: testall. Därefter kommer även konfigurationen make testall att dyka upp som ett alternativ då man väljer Run --> External Tools.

Det är troligt att ni i kommande laborationer vill återanvända makefilen. Den kan då kopieras till projektkatalogen för den nya labben och modifieras med avseende på var testfilerna finns och vad det program som skall köras heter. Därefter går det att köra de konfigurationer vi skapat enligt ovan. Konfigurationerna är genereriska genom att de bara anger att programmet /usr/bin/make skall köras med ett visst argument på en makefile som ligger i den aktuella projektkatalogen. Om ni ändrar makefilen så att den innehåller fler eller andra targets så måste ni dock skapa nya konfigurationer enligt ovan för dessa.

Observera att vi i makefilen gjort följande antaganden:

  • Testfilerna befinner sig i /usr/local/cs/edaf05/lab1. Om ni har lagt dem på annan plats får ni korrigera definitionen av TESTDIR i makefilen
  • Programmet som testas finns i klassen GS. Ändra här till ert eget namn på klassen.
  • Programmet som skall testas finns i default package i projektet. Om ni har skapat ett paket med t.ex. namnet sm för ert program så måste ni korrigera i makefilen genom att kvalificera klassnamnet med paketnamn. Om programmet som testas finns i klassen GS i paketet sm så så skriver man alltså sm.GS i java-kommandona.
